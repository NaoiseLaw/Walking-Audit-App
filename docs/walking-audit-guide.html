What Is the Walking Audit App?

A digital tool for conducting structured walkability assessments of pedestrian infrastructure — built for the Irish National Transport Authority's Universal Design Walkability Audit framework.

Purpose

The Walking Audit App enables community groups, local authorities, school travel planners, and coordinators to conduct standardised assessments of walking routes — recording GPS-tagged observations, photographic evidence, and scored assessments across 7 key categories, then generating professional PDF reports for submission to local authorities.

Who Uses It?

Four distinct user types interact with the platform at different stages of the audit lifecycle:

Auditor

Field Worker

  • Conducts on-the-ground audits
  • Logs issues with GPS + photos
  • Scores each section
  • Works offline on mobile
Coordinator

Team Lead

  • Defines walking routes
  • Assigns audits to teams
  • Reviews submissions
  • Generates reports
LA Admin

Local Authority

  • Reviews audit findings
  • Responds to recommendations
  • Tracks implementation
  • Downloads official reports
System Admin

Platform Admin

  • Full user management
  • All data access
  • System configuration
  • Analytics overview

The 7 Audit Categories

Every audit scores the walking route across seven standardised sections (matching the NTA Universal Design framework):

#SectionWhat It Covers
1FootpathsWidth, surface condition, kerb drops, obstacles, continuity
2FacilitiesSeating, shelters, public toilets, bins, water fountains
3Crossing the RoadPedestrian crossings, signal timing, tactile paving, islands
4Road User BehaviourDriver compliance, pavement parking, cycling on footpaths
5SafetyLighting, sight lines, blind spots, crime concern, traffic speed
6Look & FeelGreenery, cleanliness, street art, attractiveness, heritage
7School GatesDrop-off behaviour, pedestrian space, school zone safety (where applicable)

Walking Routes & Maps

Routes are the geographic backbone of the app. Every audit is conducted on a defined route. Google Maps provides the visual interface for browsing, selecting, and navigating routes.

What Is a Route?

A route is a defined walking path stored as a GeoJSON geometry with metadata including distance, surface type, elevation data, public transport proximity, and historical audit scores.

Route FieldDescription
Name & DescriptionHuman-readable title and description
Town / City / CountyGeographic classification
GeoJSON GeometryThe actual path coordinates (LineString)
Distance (metres)Calculated from geometry
Bounding BoxMap viewport for auto-zoom
Elevation (min/max/avg gradient)Terrain analysis for accessibility scoring
Surface TypeTarmac, cobblestone, gravel, etc.
LightingStreet lighting quality assessment
Has Public TransportFlag + distance to nearest bus/rail stop
Route TypeSchool route, leisure, commuter, etc.
Permeability ScoreHow connected the route is to the wider network
Avg ScoreMean audit score across all completed audits
Audit CountNumber of times route has been audited
Is FeaturedPinned to top of browse list

Scoring System

Each section is scored on a 1–5 scale. Scores feed into the overall audit score and are used in PDF report generation and trend analytics.

Offline Capability

ℹ️ Offline-First Design The sync_queue table acts as a buffer for all writes made without internet connectivity. When connectivity resumes, queued operations sync automatically. This allows auditors to conduct full walkthroughs in areas with no mobile signal (a common scenario on rural routes in Ireland).

Multi-User Audits

Audits support multiple participants via the audit_participants table. This allows a team of auditors to work on different sections simultaneously, with all observations consolidated under one audit record.

What the PDF Report Contains

  • Cover page — Route name, audit date, coordinator name, organisation
  • Executive summary — Overall score, key strengths and weaknesses
  • Route map — Google Maps screenshot with all issue pins
  • Section-by-section scores — Spider/radar chart of all 7 categories
  • Issue log — Full list of observations with GPS coordinates, severity, photos
  • Photo evidence — Captioned photos with location references
  • Recommendations — Prioritised action list for the Local Authority
  • Participant list — Who conducted the audit
  • Baseline comparison — Score vs. previous audits of the same route

Baseline Data & Trend Analysis

The baseline_data table stores historical measurements for each route — allowing the app to show whether conditions are improving, stable, or deteriorating over successive audits. This is the most powerful advocacy tool: showing a Local Authority that a problem was flagged 2 years ago and still isn't resolved.

How Data Will Be Shown

The long-term vision for data presentation includes:

🗺️ Public Walkability Map

A publicly accessible map (no login required) showing aggregate audit scores by street/area — a visual "walkability index" for Ireland. Users can see how their neighbourhood rates and click through to see specific issues and improvement status.

📈 Progress Dashboard for LAs

Each Local Authority gets a dedicated dashboard showing: number of audits conducted in their area, % of issues resolved, year-on-year score improvement, and a ranked list of streets most in need of improvement.

📊 National Statistics Portal

Aggregated national data showing which counties have the most walkable infrastructure, common issue types across Ireland, and trends over time. Exportable as CSV or GeoJSON for academic / government use.

📱 Citizen Reporting (Phase 4+)

A simplified public-facing version where any citizen can report a walkability issue on their street — not a full audit, but a quick pin + photo + category. These reports feed into the main database and can be promoted to full issues by coordinators.
