# Database Migrations

## Running Migrations

### Development
```bash
npm run db:migrate
```

### Production
```bash
npm run db:migrate:deploy
```

## Creating a New Migration

1. Make changes to `schema.prisma`
2. Run `npm run db:migrate`
3. Prisma will generate a new migration file
4. Review the generated SQL
5. Test the migration

## Migration Files

- Migration files are generated by Prisma
- Do not manually edit migration files
- Each migration should be reversible
- Test migrations on a copy of production data

## Notes

- PostGIS extension must be enabled in PostgreSQL
- UUID extension must be enabled
- pg_trgm extension must be enabled for text search
- Always backup database before running migrations in production
